The ECG and BP monitor
(based on the system used in Macao and Tsinghua, China)

ECG (electrocardiogram) and BP (blood pressure) telemonitoring on a medical level are closely related, because they both concern blood flow. In a telemonitoring system they are measured separately, but will be considered in parallel.

The patient’s side of the system

The patient’s unit was originally a PC workstation. However, many patients were unfamiliar with computers as well as unable to afford the system and its installation cost. This was rather unfortunate as the software had been programmed for PCs. To simplify the system without rendering the software useless, the improved version would contain a PC motherboard in addition to a hard disk and modem. It would behave as a computer, but wouldn’t have any unnecessary extras. The patient could rent it, thereby saving cost and its setup was much simpler.

The cardiac activity data is acquired by use of strategically placed electrodes connected to a wireless transmitter that fits neatly into a patient’s pocket. An antenna on the patient’s unit detects the data. That way, the patient isn’t restricted in daily activity and the monitoring is comprehensive.

The BP monitor is a watch which closely resembles the current instrument. The patient takes his or her pulse at suggested times.

The display panel is an LCD monitor which requires comparatively little power, saving operational cost.

The unit has 5 modes:

  • Initialization – instructions are given for the user to put on the ECG-monitoring electrodes. The system alerts the patient whether they have been placed correctly, though it also learns his or her placement habits and adjusts the reading magnitudes accordingly. It then switches to online monitoring mode.


  • ECG Monitoring (Online) – the ECG is displayed, and if the readings stray away from normal, data is automatically transferred to the hospital monitoring center so that the staff are notified as soon as possible of any potential problems. If the staff wish to communicate with the patients, a message can be sent and displayed on the LCD monitor.


  • ECG Monitoring (Manual) – In the event of patient discomfort or at the request of the physician (e.g. for checkup or advice), data can be force-transferred by switching to this mode (pressing the ‘dial’ button illustrated above). The user’s feelings can also be conveyed by an emoticon which appears on the monitoring center’s display window.


  • Holter Mode - For comprehensive analysis of a patient’s well being, the monitor can be set to record the ECG pattern over a long period (i.e. days). The memory consumption isn’t really an issue with modern hard disk capacities, though daa compression is employed to reduce the cost of transferring the data to the hospital.


  • BP Mode - the blood pressure is recorded, under guidance from the system, and transferred to the monitoring center.

The monitoring center’s side of the system

Usually located within a hospital, the monitoring station in this particular system runs under UNIX; the software has been produced with the C language.

The software has two major components, the real time monitoring system (RTMS) and the memory information management system (MIMS).

The RTMS is responsible for receiving and monitoring incoming data from multiple patients, as well as saving the information in files (placing these files in the correct location is not its job).

The MIMS has 4 components, two of which are for analysis of the data which has been saved, and the other two of which are for storing all the data.

  • The ECG analyzer uses various algorithms to detect abnormal ECG paterns such as asystole, missed beats, ventricular ectopic beats, atrial premature beats, bradycardia, tachycardia. They are flagged on the monitoring display to attract the medical personnel’s attention. There is also a noise detection and removal algorithm.


  • The BP analyzer displays the systolic and diastolic pressures (the two heart beats), average pressure, heart rate, and the time of the data recording. It can also plot a graph showing the change in any of these over a period of time.


  • The file manager is responsible for reading from and writing to 3 databases. The patient profile database contains the patient id, test times, the path where all his or her data files are stored as well as some obvious personal details. The ECG and BP databases contain the complete set of data files for the respective measurements taken from a particular patient.


  • The backup tool contains 4 functional modules which:
    o list the data to be backed up
    o transfer the data to magnetic tape
    o list the data contained on any one magnetic tape
    o retrieve any requested data.
