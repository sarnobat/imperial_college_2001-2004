Developer Information

Introduction

The contents of this page apply if you are interested in the internals of the DB2LaTeX XSL transformation stylesheets, or if you are interested in becoming an active developer and mantainer of the DB2LaTeX stylesheets, or just if you would like to be able to write your own XSL customization layers.

Getting the XSL stylesheets

Getting a stable distribution

DB2LaTeX releases can be downloaded from the SourceForge web site and its mirrors http://www.sourceforge.net/projects/db2latex

Getting CVS snapshot

Another possibility is to download a snapshot of the CVS repository. Unless you are a DB2LaTeX developer, only anonymous (read-only) CVS access is allowed.

Anonymous CVS Access

Anonymous CVS read-only access is available for the DB2LaTeX project. Follow the following instructions:

baldrick(/opt/sgml/xsl)% cvs -z3
      -d:pserver:anonymous@cvs.db2latex.sourceforge.net:/cvsroot/db2latex
      login baldrick(/opt/sgml/xsl)% cvs -z3
      -d:pserver:anonymous@cvs.db2latex.sourceforge.net:/cvsroot/db2latex co
      db2latex baldrick(/opt/sgml/xsl)% cvs -z3
      -d:pserver:anonymous@cvs.db2latex.sourceforge.net:/cvsroot/db2latex
      logout (Logging out of anonymous@cvs.db2latex.sourceforge.net)

Developer CVS Access

Developer (full) CVS access is available for the DB2LaTeX project upon request.

baldrick(/opt/sgml/xsl)% export CVS_RSH=ssh
      baldrick(/opt/sgml/xsl)% cvs -z3
      -d:ext:rcasellas@cvs.db2latex.sourceforge.net:/cvsroot/db2latex co
      db2latex rcasellas@cvs.db2latex.sourceforge.net's password: xxxxxyxxxx
      cvs server: Updating db2latex...
Important

Do not forget to export (setenv in C-sh) the CVS_RSH variable. Otherwise, sourceforge will close the connection without further notice. (Yes, it happened to me too :)
