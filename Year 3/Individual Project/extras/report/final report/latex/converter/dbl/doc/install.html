Installing the Stylesheets

How? (Using lynx)

The DB2LaTeX stylesheets no longer depend on Norman DocBook XSL stylesheets. You just need to checkout the latest (and greatest?) release from the cvs repository. In a Unix/cygwin like environment with /bin/sh and lynx installed, you may type :

      
you@yourhost(/usr/any/path)%  lynx -source http://db2latex.sourceforge.net/get-db2latex.sh | sh
###############################################################
get-db2latex: 
###############################################################
...

And that's all. The directory db2latex will be created, and you only need to type the following command to update your local copy:

      
you@yourhost(/usr/any/path)%  cd db2latex
you@yourhost(/usr/any/path/db2latex)%  cvs update

How? (Using CVS snapshots)

Alternatively, you may want to checkout the cvs repository manually :

      
baldrick(/home/casellas/sgml/xsl)% cvs -z3 -d:pserver:anonymous@cvs.db2latex.sourceforge.net:/cvsroot/db2latex login
baldrick(/home/casellas/sgml/xsl)% cvs -z3 -d:pserver:anonymous@cvs.db2latex.sourceforge.net:/cvsroot/db2latex co db2latex
baldrick(/home/casellas/sgml/xsl)% cvs -z3 -d:pserver:anonymous@cvs.db2latex.sourceforge.net:/cvsroot/db2latex logout

How? (Downloading releases)

What If I don't have cvs / lynx / sh installed? You may download releases from the Sourceforge DB2LaTeX page. Then :

  • Download the file db2latex-x.y.z.tar.gz

  • Unpack ( tar xvfz db2latex-x.y.z.tar.gz) the distribution. the db2latex directory hierarchy is created.

you@yourhost $ mkdir /tmp/downloads you@yourhost $ cd
    /tmp/downloads (download file) you@yourhost $ su Password: root@yourhost
    # cp /tmp/downloads/db2latex-x.y.z.tar.gz /usr/fianl/destination
    root@yourhost # cd /usr/fianl/destination root@yourhost # tar xvfz
    db2latex-x.y.z.tar.gz (creates the directory docbook/latex)

See the README file and the xsl/sample subdirectory for examples. I am currently in the process of Documenting the stylesheets the same way Norman does,using the doc:namespace. Documentation is just in its barebones, but I plan to improve it.

Copyright and Disclaimer

In order to make them more portable, the test XML files specify the docbookx DTD by declaring a SYSTEM identifier that points to the Oasis URL. This works but affects performance. Change the xml DOCTYPE declarations to reflect your local installation.
