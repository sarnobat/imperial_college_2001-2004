Using DB2LaTeX

Introduction

In this section we will give the basic steps to use the DB2LaTeX XSL Stylesheets.

  • SGMLBASE corresponds to your SGML root directory. (e.g. /opt/sgml or /usr/lib/sgml)

The XSLT processor

To transform a XML document using XSL stylesheets you need an XSLT processor. There are several commercial/free XSLT processors available. The installation may vary according to the processor used. Most processors are written in C++/Java/Perl. In the following sections we will describe hos to use the stylesheets using Java Xalan. Former versions of the stylesheets used to work with XT, but since at the time being the DB2LaTeX XSL stylesheets still depend on the Norman Walsh stylesheets and he is considering dropping support for XT, DB2LaTeX releases are tested using Xalan for Java Version 2.

Basically, all XSLT processors start with three parameters: The input XML document, the stylesheet to apply and the output filename. The syntaxis depends on your processor.

Apache Xalan XML

Xalan is a Java XSLT processor developed by the Apache-XML team (the Apache XML Project). There is currently a Perl version, a Java version and a C++ version. Xalan-J2 is known to work.

Applying the stylesheets to yor XML document

Command Line Interface

java -cp
      $(SGMLBASE)/java/xalan.jar:$(SGMLBASE)/xerces.jar:$(SGMLBASE)l/java/bsf.jar
      \ org.apache.xalan.xslt.Process \ -VALIDATE OFF \ -IN document.xml \
      -XSL /path/to/db2latex/xsl/docbook.xsl \ -OUT output.tex
